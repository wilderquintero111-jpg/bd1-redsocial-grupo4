# Tarea 2 Modelo Lógico de Base de Datos
Red Social Estudiantil Pascualina
Diseño de una base de datos con estándares y buenas prácticas

En el desarrollo de sistemas de información es fundamental contar con una adecuada estructura de almacenamiento de datos que garantice organización, eficiencia y consistencia en la información. En este contexto, las bases de datos juegan un papel esencial al permitir modelar y gestionar los datos de acuerdo con las necesidades de una aplicación o sistema.

La presente actividad tiene como objetivo diseñar un modelo lógico de base de datos para la Red Social Estudiantil Pascualina, partiendo del modelo conceptual previamente elaborado. Para lograrlo, se aplica el proceso de normalización de bases de datos, utilizando las tres primeras formas normales: Primera Forma Normal (1FN), Segunda Forma Normal (2FN) y Tercera Forma Normal (3FN). Este proceso permite organizar los datos de manera eficiente, reducir redundancias y garantizar la integridad de la información.

Durante el desarrollo de la tarea, el equipo realizó un análisis de las necesidades del sistema, identificando las principales acciones que los usuarios podrán realizar dentro de la red social, como crear perfiles, interactuar con otros usuarios, compartir contenido y participar en diferentes actividades dentro de la plataforma. A partir de este análisis se desarrolló un proceso de lluvia de ideas y construcción de un mapa conceptual, el cual permitió identificar las posibles entidades, atributos y relaciones necesarias para el sistema.

Posteriormente, se llevó a cabo el modelado lógico de la base de datos, transformando las entidades del modelo conceptual en tablas y definiendo sus respectivos atributos, claves primarias, claves foráneas y restricciones necesarias para asegurar la correcta organización de los datos. Asimismo, se elaboró un diccionario de datos, donde se describen los elementos principales de cada tabla, incluyendo el nombre de los atributos, tipo de dato, longitud, descripción y justificación de su uso dentro del sistema.

Finalmente, se presentan los resultados obtenidos a través del proceso de normalización, el diseño del modelo lógico de la base de datos, y la documentación correspondiente que explica las decisiones tomadas durante el análisis y diseño del sistema de información.

Objetivo de la tarea

Diseñar un modelo lógico de base de datos para la Red Social Estudiantil Pascualina aplicando el proceso de normalización (1FN, 2FN y 3FN), con el fin de organizar adecuadamente la información del sistema y garantizar la integridad y eficiencia en la gestión de los datos.

Actividades desarrolladas

Para el desarrollo de esta tarea se realizaron las siguientes actividades:

Análisis de necesidades:
Cada integrante del equipo revisó las necesidades planteadas para la Red Social Pascualina y las funcionalidades que los usuarios deben poder realizar dentro del sistema.

Lluvia de ideas y mapa de conceptos:
Se elaboró un mapa conceptual inicial donde se identificaron las posibles entidades, atributos y relaciones del sistema.

Modelado lógico y normalización:
Se aplicó el proceso de normalización mediante las formas normales 1FN, 2FN y 3FN, con el fin de estructurar adecuadamente las tablas del modelo lógico.

Elaboración del diccionario de datos:
Se documentaron los elementos principales del modelo lógico, incluyendo:

Claves primarias (PK)

Claves foráneas (FK)

Claves únicas

Tipos de datos

Descripción de cada atributo

Presentación y retroalimentación:
Cada miembro del equipo presentó su propuesta de normalización y se realizaron ajustes con base en la retroalimentación obtenida.

Versión final del modelo:
Se integraron los resultados y se elaboró la versión final del modelo lógico junto con la documentación correspondiente.
